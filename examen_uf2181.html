Preguntas y Respuestas: Testing de Software

1. ¿Qué es el software testing?
Es el proceso de evaluar un sistema para identificar defectos y asegurar que cumple con los requisitos especificados.
2. ¿Qué tipos de errores se pueden detectar?
Errores de programación, fallos de lógica, problemas en la documentación o desviaciones de la calidad esperada.
3. ¿Qué significa que no haya errores?
Significa que no se han encontrado defectos durante las pruebas, aunque no garantiza que el software sea perfecto.
4. ¿Qué son la validación y la verificación?
Validación asegura que el software cumple con las necesidades del usuario. Verificación confirma que el producto fue construido correctamente según especificaciones.
5. ¿Qué son las pruebas de caja negra?
Pruebas que evalúan la funcionalidad sin conocer la estructura interna. Se basan en entradas y salidas.
6. ¿Qué se prueba en una interfaz?
La usabilidad, coherencia, accesibilidad, y funcionamiento correcto de los elementos interactivos.
7. ¿Qué son las pruebas de estrés?
Son pruebas que evalúan el comportamiento del sistema bajo condiciones extremas de carga.
8. ¿Quiénes son los usuarios beta?
Usuarios reales que prueban el software antes del lanzamiento final para detectar errores no previstos.
9. ¿Qué es una versión Alpha?
Es una versión temprana del software, con pruebas internas, antes de ser liberada para pruebas beta.
10. ¿Qué es el RTM (Release To Manufacturing)?
Es la versión final del software lista para su distribución o producción.
11. ¿Qué significa un caso de prueba sin errores?
Es una prueba que se ejecutó correctamente sin producir fallos.
12. ¿Qué es un caso de prueba?
Es un conjunto de condiciones y pasos definidos para verificar una funcionalidad específica del software.
13. ¿Qué evalúa el rendimiento?
Evalúa la eficiencia del sistema y la experiencia del usuario bajo diferentes condiciones de uso.
14. ¿Qué es una prueba de carga?
Es una prueba que mide el rendimiento del sistema al manejar un volumen esperado de usuarios o transacciones.
15. ¿Qué implica una prueba de estrés?
Implica llevar el sistema más allá de sus límites para observar cómo se comporta y si colapsa.
16. ¿Qué es un archivo JAR en Java?
Es un archivo que empaqueta múltiples clases y recursos de Java en un solo contenedor ejecutable.
17. ¿Es la documentación una etapa del ciclo de vida?
No, la documentación no es una etapa por sí sola, pero es transversal y necesaria en todas las fases del desarrollo.
18. ¿Qué implica implementar un software?
Implica codificar, integrar y poner en funcionamiento el sistema según los requerimientos definidos.
19. ¿Qué tipos de pruebas existen según el tipo de usuario?
Pruebas de usuario final, pruebas técnicas y pruebas de instalación.
20. ¿El usuario debe asistir a todas las pruebas?
No necesariamente. El usuario participa en validaciones clave, pero no está obligado a asistir a todas las pruebas técnicas.
