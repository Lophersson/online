30 Preguntas sobre Arquitectura de Capas en C#

1. ¿Qué es una arquitectura de capas en C#?

Es un patrón de diseño que separa la lógica de una aplicación en distintas capas para mejorar su organización y mantenimiento.

2. ¿Cuáles son las capas más comunes en una arquitectura de C#?

Entidad, Datos, Negocio y Presentación.

3. ¿Qué responsabilidad tiene la capa de entidad?

Define las clases y modelos de datos que representan objetos del dominio.

4. ¿Cuál es el propósito de la capa de datos?

Encapsula el acceso a la base de datos, realizando operaciones CRUD.

5. ¿Qué hace la capa de negocio?

Implementa la lógica del negocio, validaciones y procesamiento de reglas.

6. ¿Para qué sirve la capa de presentación?

Interactúa con el usuario, mostrando datos y recogiendo entradas.

7. ¿Qué es un DTO?

Un Data Transfer Object, usado para transportar datos entre capas sin exponer entidades.

8. ¿Qué es un repositorio en la capa de datos?

Una clase que actúa como intermediario entre la base de datos y la lógica de negocio.

9. ¿Qué ventajas tiene esta arquitectura?

Modularidad, mantenimiento, pruebas unitarias y separación de responsabilidades.

10. ¿Puede una capa acceder directamente a otra?

Sí, pero se recomienda seguir el flujo: Presentación → Negocio → Datos → Entidad.

11. ¿Qué herramienta ORM se usa comúnmente en C#?

Entity Framework.

12. ¿Qué significa CRUD?

Create, Read, Update, Delete.

13. ¿Puede la capa de presentación contener lógica de negocio?

No, solo debe encargarse de mostrar datos y capturar entradas del usuario.

14. ¿Qué lenguaje se usa en la capa de presentación web en C#?

HTML, CSS, JavaScript y Razor (en ASP.NET).

15. ¿Qué patrón de diseño se suele usar junto con esta arquitectura?

MVC (Modelo Vista Controlador).

16. ¿Qué archivo define la conexión a la base de datos en un proyecto C#?

App.config o Web.config.

17. ¿Qué tipo de validaciones se hacen en la capa de negocio?

Validaciones de reglas del negocio como límites, consistencia y cálculos.

18. ¿Puede la capa de datos lanzar excepciones?

Sí, pero deben ser controladas por la capa de negocio.

19. ¿Qué es una interfaz en C#?

Un contrato que define métodos que una clase debe implementar.

20. ¿Qué beneficios trae el uso de interfaces entre capas?

Flexibilidad, inyección de dependencias y facilidad de pruebas.

21. ¿Qué biblioteca permite la inyección de dependencias en C#?

Microsoft.Extensions.DependencyInjection.

22. ¿Qué es una entidad en Entity Framework?

Una clase que representa una tabla de la base de datos.

23. ¿Qué responsabilidad tiene un controlador en ASP.NET MVC?

Recibe peticiones del usuario, llama a la lógica de negocio y retorna una vista.

24. ¿Qué es un servicio en la capa de negocio?

Una clase que encapsula reglas de negocio y operaciones de dominio.

25. ¿Cómo se evita el acoplamiento entre capas?

Usando interfaces y principios SOLID.

26. ¿Qué significa el principio de responsabilidad única?

Una clase debe tener una única razón para cambiar.

27. ¿Qué es un ViewModel?

Una clase usada para estructurar datos específicos que se muestran en la vista.

28. ¿Qué significa desacoplar capas?

Evitar que una capa dependa directamente de la implementación de otra.

29. ¿Cuál es la desventaja de una arquitectura de capas?

Puede generar sobrecarga en proyectos muy pequeños o simples.

30. ¿Es posible usar APIs REST con esta arquitectura?

Sí, implementando controladores REST en la capa de presentación.
